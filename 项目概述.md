# Claude技能库项目概述

## 1. 项目简介

Claude技能库是一个用于增强Claude AI能力的技能集合，这些技能可以动态加载，使Claude在特定任务上表现更出色。技能可以教会Claude如何以可重复的方式完成特定任务，无论是使用公司品牌指南创建文档、使用组织特定工作流分析数据，还是自动化个人任务。

## 2. 项目类型

这是一个**AI技能库项目**，为Anthropic的Claude AI提供各种功能扩展，涵盖多个领域和应用场景。

## 3. 项目结构

项目采用清晰的模块化结构，主要包含以下目录：

| 目录/文件 | 描述 |
|---------|------|
| `/skills` | 包含所有技能实现，按功能和领域分类 |
| `/spec` | 技能规范（已迁移到外部网站） |
| `/template` | 技能创建模板 |
| `/README.md` | 项目核心说明文档 |
| `/THIRD_PARTY_NOTICES.md` | 第三方库许可证说明 |

## 4. 技能分类

技能库中的技能可以分为以下几大类别：

### 4.1 文档处理技能
- **docx**: 全面的文档创建、编辑和分析，支持跟踪更改、评论、格式保留和文本提取
- **pdf**: PDF文件的创建、编辑和分析
- **pptx**: PowerPoint演示文稿的创建和编辑
- **xlsx**: Excel电子表格的处理

### 4.2 创意设计技能
- **algorithmic-art**: 算法艺术生成
- **brand-guidelines**: 品牌指南遵循
- **canvas-design**: 画布设计创作
- **theme-factory**: 主题工厂，提供多种设计主题

### 4.3 开发技术技能
- **webapp-testing**: Web应用测试
- **web-artifacts-builder**: Web构件构建
- **mcp-builder**: MCP服务器生成

### 4.4 企业通信技能
- **doc-coauthoring**: 文档协作
- **internal-comms**: 内部通信模板
- **slack-gif-creator**: Slack GIF创建器

## 5. 技能组成

每个技能通常包含以下组件：

1. **SKILL.md**: 技能的核心定义文件，包含：
   - YAML前置元数据（名称、描述、许可证等）
   - 使用说明和指南
   - 工作流程和最佳实践

2. **脚本文件**: 用于执行具体操作的代码（Python、JavaScript、Shell等）

3. **资源文件**: 支持技能运行的资源（字体、模板、schema等）

4. **许可证文件**: 技能的许可证信息

## 6. 使用方式

### 6.1 Claude Code

```bash
# 添加市场
/plugin marketplace add anthropics/skills

# 安装特定技能集
/plugin install document-skills@anthropic-agent-skills
/plugin install example-skills@anthropic-agent-skills
```

### 6.2 Claude.ai

部分技能已内置到Claude.ai的付费计划中。要使用自定义技能，可按照官方指南上传。

### 6.3 Claude API

通过Claude API可以使用Anthropic预构建的技能，也可以上传自定义技能。

## 7. 许可证说明

- 许多技能采用Apache 2.0开源许可证
- 文档创建和编辑技能（docx、pdf、pptx、xlsx）为专有许可证
- 每个技能目录下的LICENSE.txt文件包含完整的许可证条款

## 8. 项目愿景

该项目旨在展示Claude技能系统的可能性，为开发者提供灵感和参考，帮助他们创建自己的定制技能，以满足特定需求。

## 9. 贡献指南

虽然项目主要由Anthropic维护，但欢迎开发者参考这些技能实现，创建自己的自定义技能。创建技能的基本步骤：

1. 创建一个新目录
2. 编写SKILL.md文件，包含技能定义和使用说明
3. 添加必要的脚本和资源文件
4. 测试技能功能
5. 部署和使用

## 10. 注意事项

- 这些技能仅供演示和教育目的使用
- Claude实际提供的实现和行为可能与示例不同
- 在关键任务中使用前，请在自己的环境中彻底测试技能

# 技能开发指南

## 1. 技能基础结构

每个技能必须包含一个`SKILL.md`文件，位于技能的根目录。该文件采用Markdown格式，包含YAML前置元数据和技能说明内容。

### 1.1 YAML前置元数据

```yaml
---
name: skill-name          # 技能唯一标识符（小写，用连字符分隔空格）
description: "技能描述，说明技能用途和使用场景"  # 清晰描述技能功能和适用情况
license: Apache 2.0       # 可选，技能许可证
---
```

### 1.2 技能内容结构

技能内容应包含以下部分（根据技能类型和复杂度调整）：

1. **概述**: 简要介绍技能用途和功能
2. **工作流程**: 说明技能使用的工作流程和决策树
3. **使用指南**: 详细说明如何使用技能
4. **代码示例**: 提供技能使用的代码示例
5. **最佳实践**: 分享使用技能的最佳实践
6. **依赖项**: 列出技能所需的依赖项

## 2. 技能开发步骤

### 2.1 确定技能需求

- 明确技能的核心功能和目标
- 确定技能的使用场景和用户群体
- 分析技能所需的资源和依赖

### 2.2 创建技能目录结构

```
skills/skill-name/
├── SKILL.md           # 技能核心定义文件
├── LICENSE.txt        # 许可证文件
├── scripts/           # 脚本目录
│   ├── script1.py
│   └── script2.js
└── resources/         # 资源目录
    ├── template.txt
    └── image.png
```

### 2.3 编写SKILL.md文件

- 填写YAML前置元数据
- 编写技能说明和使用指南
- 包含工作流程和最佳实践
- 提供清晰的代码示例

### 2.4 实现技能脚本

- 根据技能需求选择合适的编程语言
- 编写高效、可靠的脚本代码
- 确保脚本具有良好的错误处理
- 提供必要的注释和文档

### 2.5 添加资源文件

- 包含技能所需的所有资源
- 确保资源文件的许可证合规
- 优化资源文件大小（如适用）

### 2.6 测试技能

- 在开发环境中测试技能功能
- 验证技能在各种场景下的表现
- 检查技能的错误处理和边界情况
- 测试技能的性能和可靠性

### 2.7 部署和发布

- 按照官方指南部署技能
- 提供技能的使用文档和示例
- 定期更新和维护技能

## 3. 技能开发最佳实践

### 3.1 命名规范

- 技能名称应简洁、描述性强

### 3.4 代码质量

- 编写清晰、可维护的代码

### 3.5 文档质量

- 提供全面、准确的文档
- 使用清晰的结构和格式

- 定期更新文档以反映技能变化

# 技能使用指南

## 1. 技能选择

根据任务需求选择合适的技能：

### 1.1 文档处理
- **docx**: 处理Word文档，支持创建、编辑、跟踪更改等
- **pdf**: 处理PDF文件，支持表单填写、文本提取等
- **pptx**: 处理PowerPoint演示文稿
- **xlsx**: 处理Excel电子表格

### 1.2 创意设计
- **algorithmic-art**: 生成算法艺术
- **canvas-design**: 创建画布设计作品
- **theme-factory**: 生成设计主题

### 1.3 开发技术
- **webapp-testing**: 测试Web应用
- **web-artifacts-builder**: 构建Web构件

### 1.4 企业通信
- **brand-guidelines**: 遵循品牌指南
- **internal-comms**: 创建内部通信内容

## 2. 技能使用流程

### 2.1 基本使用流程

1. **识别需求**: 明确任务目标和要求
2. **选择技能**: 根据需求选择合适的技能
3. **阅读文档**: 仔细阅读技能的SKILL.md文件，了解使用方法和最佳实践
4. **执行操作**: 按照文档说明执行技能操作
5. **验证结果**: 检查技能输出是否符合预期
6. **优化调整**: 根据需要调整参数或流程

### 2.2 高级使用技巧

- **组合技能**: 根据需要组合使用多个技能
- **自定义参数**: 根据任务需求调整技能参数
- **批处理操作**: 对多个文件或任务执行批处理
- **自动化工作流**: 将技能集成到自动化工作流中

## 3. 常见问题和解决方案

### 3.1 技能未按预期工作

**解决方案**:
- 检查技能文档，确保使用方法正确
- 验证输入参数是否符合要求
- 检查依赖项是否正确安装
- 查看错误日志以定位问题

### 3.2 技能执行速度慢

**解决方案**:
- 优化输入数据，减少处理量
- 调整技能参数以提高性能
- 考虑使用批处理方式减少重复操作
- 检查系统资源使用情况

### 3.3 技能输出格式不符合要求

**解决方案**:
- 查看技能文档，了解输出格式选项
- 调整技能参数以改变输出格式
- 使用其他工具转换输出格式
- 联系技能开发者寻求帮助

## 4. 最佳实践

### 4.1 技能使用前

- 充分了解技能的功能和局限性
- 阅读完整的技能文档和示例
- 测试技能在类似场景下的表现
- 准备好所需的输入数据和资源

### 4.2 技能使用中

- 按照文档说明正确使用技能
- 监控技能执行过程和资源使用
- 及时处理技能产生的错误和警告
- 记录技能使用过程和结果

### 4.3 技能使用后

- 验证输出结果是否符合预期
- 保存技能执行日志和结果
- 评估技能的效果和效率
- 提供反馈和建议，帮助改进技能
